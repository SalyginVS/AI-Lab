# 47 Heredoc в интерактивном терминале ненадёжен для длинных файлов

## Симптом

При создании markdown-артефактов через `cat > file << 'EOF'` содержимое обрывается на середине.

## Причина

Интерактивный терминал + длинные heredoc → проблемы с буфером ввода. Особенно при copy-paste из ChatGPT.

## Решение

Для длинных файлов использовать:
- `printf` с escape-строками
- Python one-liner (`python3 -c "..."`)
- Редактор (`nano`, `vim`)
- `scp` готового файла с клиента
- Не heredoc в интерактивном режиме
