# Этап 8C — Orchestrator PoC

## Цель

Реализовать [[orchestrator.py]] — headless Python-скрипт на сервере Ubuntu, выполняющий последовательный multi-model pipeline через [[gateway.py]]. Первый компонент слоя 4 (Orchestration/Automation) [[Целевая архитектура v1.4]].

## Что сделано

Реализованы 5 pipeline в [[pipelines.yaml]]:

| Pipeline | Модели |
|----------|--------|
| `smoke` | qwen3.5:9b |
| `execute-review` | qwen3-coder:30b → deepseek-r1:32b |
| `plan-execute-review` | qwen3:30b → qwen3-coder:30b → deepseek-r1:32b |
| `review-only` | deepseek-r1:32b |
| `docs-generate` | qwen3-coder:30b |

Все pipeline отработали на реальных задачах. Результаты сохраняются в `~/llm-gateway/results/YYYYMMDD_HHMMSS.json`.

## Реальные замеры (plan-execute-review)

| Task | Planner | Executor | Reviewer | Total |
|------|---------|----------|----------|-------|
| Simple | 45.25 сек | ~6 сек | ~37 сек | **88.49 сек** |
| Medium | 11.80 сек | 4.08 сек | 37.46 сек | **53.34 сек** |
| Complex | 26.63 сек | 5.01 сек | 35.02 сек | **66.66 сек** |

## Ключевые решения

- [[ADR-001 Planner qwen3:30b вместо qwen3.5:35b]] — qwen3.5:35b нестабилен при загрузке в Ollama 0.18.0
- Фильтрация `<think>...</think>` на шаге planner перед передачей в executor
- `temperature: 0.2` для executor/reviewer, `0.4` для planner; `num_ctx: 8192`
