# ADR-013: Text-to-SQL PoC — Evaluator separation и остановка prompt micro-tuning

## Контекст

Stage 11A — первый PoC паттерна NL→SQL. В ходе работы возникли два архитектурных вопроса:

1. Evaluator для NL-to-SQL benchmark смешивался с model/prompt layer, искажая метрики.
2. Prompt engineering дал +15–20 п.п. SA, но дальнейшие точечные правки вызывали regressions (prompt bleed).

## Решение

**Evaluator separation:** evaluator (comparator + acceptance rules) выделен в отдельный версионируемый компонент (v1 → v2). Baseline truth фиксируется на конкретной версии evaluator и не пересматривается при изменении prompt layer.

**Prompt micro-tuning freeze:** после достижения SA 85–90% дальнейшая оптимизация prompt addendum прекращена. Следующий слой улучшений — semantic grounding / retrieval-assisted SQL (Vanna, few-shot retrieval по schema), а не расширение монолитного system prompt.

## Альтернативы

1. Продолжать prompt tuning до 95%+ → отклонено: regressions, coupling, не масштабируется.
2. Ослабить evaluator для повышения SA → отклонено: теряется baseline truth.

## Последствия

- Baseline truth (v2) сохранён как reference point для будущих экспериментов.
- v3 results (SA 85–90%) — доказательство value prompt engineering, но не новый frozen baseline.
- Semantic layer запланирован как следующий шаг (не привязан к конкретному этапу roadmap).
