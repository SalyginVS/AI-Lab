### 15-3. 128K Context Window — Blocked/External ⚠️

**Результат:** Gemma 4 long-context (65K+) не работает на текущем стеке. Это **upstream bug в Ollama 0.20.0 / llama.cpp**, а не ограничение оборудования лаборатории.

**Эксперименты проведены:**

| Модель | num_ctx | Загрузка | Runner start | Генерация | Классификация |
|--------|---------|----------|-------------|-----------|---------------|
| gemma4:31b | 131072 | ✅ | ✅ | ❌ stall | timeout/stall |
| gemma4:31b | 65536 | ✅ | ✅ | ❌ stall | stall after load |
| gemma4:26b | 131072 | ✅ | ✅ | ❌ stall | stall after load |
| gemma4:26b | 65536 | ✅ (weak) | ✅ (weak) | ❌ hang | likely unstable |

**Характерная картина:** модель загружается в GPU (VRAM ~20–23 ГБ), runner стартует за 4–8 сек, но GPU utilization = 0% — генерация не начинается.

**Root cause (F — подтверждено по открытым issues):**

Gemma 4 использует Sliding Window Attention (SWA) с гибридной архитектурой памяти. llama.cpp (upstream для Ollama) не полностью поддерживает context-shift для этой архитектуры — при больших контекстах происходит «forcing full prompt re-processing due to lack of cache data» (llama.cpp #21379), что приводит к stall.

Проблема воспроизводится на:
- RTX 5090 (Ollama #15237)
- RTX 3090 multi-GPU (Ollama #15284)
- DGX Spark / Blackwell GB10 (Ollama #15318)
- M1 Max 64 ГБ (Ollama #15286)
- ROCm (llama.cpp #21416)

**Вывод:** hardware-agnostic bug. Лаборатория работает корректно; ограничение — upstream.

## Грабли

### #43. Gemma 4 long-context stall в Ollama 0.20.0

**Симптом:** gemma4:31b и gemma4:26b при num_ctx ≥ 65536 загружаются, runner стартует, но генерация не начинается. GPU utilization = 0%.

**Причина:** upstream bug в llama.cpp — Sliding Window Attention (SWA) архитектура Gemma 4 ломает context-shift механизм. При больших контекстах система пытается переобработать весь промпт, что приводит к stall.

**Статус:** Blocked/External. Отслеживать: Ollama releases 0.20.1+, llama.cpp issue #21379.

**Workaround:** Использовать Gemma 4 с ctx ≤ 32768 (стабильно). Для long-context задач — non-Gemma модели (qwen3-coder, deepseek-r1).

### #44. Gateway MAX_NUM_CTX потолок 32768

**Симптом:** gateway отклоняет запросы с num_ctx > 32768 (validation error).

**Причина:** Жёстко заданный потолок `MAX_NUM_CTX = 32768` в `gateway/models.py`.

**Решение:** Поднять до 131072. Применено в ходе этапа 15.

### #45. Gateway HTTPX_TIMEOUT 600s недостаточен для long-context

**Симптом:** gateway прерывает запрос по timeout при длительном prompt processing.

**Причина:** `HTTPX_TIMEOUT = 600.0` (10 мин) в `gateway/__init__.py` — недостаточно для cold-start + long prompt eval.

**Решение:** Поднять до 3600.0 (1 час). Применено в ходе этапа 15.
