# ADR-008 — Модуляризация gateway.py в Python-пакет

## Статус

**Закрыт.** Выполнен в Этапе 10A (2026-03-31).

## Контекст

gateway.py v0.7.0 — 994 строки. По roadmap предстояло добавить: /v1/embeddings (9A), structured logging (10A), /metrics (10B), /v1/orchestrate (16). Прогноз — рост до 2000+ строк. Монолитный файл станет неуправляемым.

## Решение (принято 2026-03-20)

Разбить gateway.py на Python-пакет `gateway/` при первом расширении (не позднее этапа 10A). Не отдельные сервисы (overkill для лаборатории), а модули в одном процессе — один systemd unit, один порт.

**Дедлайн:** этап 10A. Не откладывать дальше.

## Реализация (2026-03-31, этап 10A)

Монолит gateway.py v0.8.0 (1305 строк) разбит на пакет `gateway/` с 8 модулями:

| Модуль | Строк | Назначение |
|--------|-------|-----------|
| `__init__.py` | 113 | VERSION, константы |
| `logging_config.py` | 106 | JSONFormatter, setup_logging() |
| `models.py` | 135 | Pydantic/dataclass модели |
| `errors.py` | 69 | GatewayError + exception handlers |
| `upstream.py` | 212 | httpx client, retry, error classification |
| `chat.py` | 595 | /v1/chat/completions |
| `embeddings.py` | 277 | /v1/embeddings |
| `listing.py` | 108 | /v1/models, /health |
| `app.py` | 153 | FastAPI assembly |

Entrypoint: `run.py` (25 строк). Итого: 1793 строки.

**Отклонения от плана в архитектуре v1.5:**
- `router.py` и `streaming.py` не выделены — вся chat-логика в `chat.py` (595 строк, управляемо)
- `auth.py` не выделен — middleware в `app.py` (будет вынесен при multi-user, этап 14)
- `metrics.py` и `orchestrator.py` — будут добавлены в этапах 10B и 16

Иерархия импортов без циклов: `__init__` → `logging_config` → `models` → `errors` → `upstream` → `chat/embeddings/listing` → `app`.

## Последствия

- Добавление нового endpoint = новый модуль + `app.include_router()`. Не затрагивает существующие.
- При переносе на Enterprise — модули можно вынести в отдельные сервисы, если нагрузка потребует.
- Тестирование изолировано по модулям.
