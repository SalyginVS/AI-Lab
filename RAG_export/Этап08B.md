# Этап 08B — Terminal Policy Framework

## Задача
Обеспечить корректное поведение Continue Agent при выполнении команд в Windows PowerShell.

## Результат
Agent использует PowerShell-native команды вместо bash-style. Кириллические имена файлов читаются корректно. Правила зафиксированы глобально через `01-general.md`.

## Что сделано

### Диагностика
Обнаружено два дефекта: Agent генерирует `ls -la` вместо `Get-ChildItem`; при выводе файлов с кириллическими именами PowerShell показывает кракозябры.

### Реализация через global rule
Добавлен Windows-блок в `%USERPROFILE%\.continue\rules\01-general.md` (`alwaysApply: true`):

- PowerShell-native команды: `Get-ChildItem`, `Get-Content`, `Where-Object`, `Select-String`
- Self-contained execution — каждая команда независима
- Не создавать временные файлы без запроса
- Не оборачивать всю команду в строковые кавычки
- UTF-8 preamble для non-ASCII имён файлов

### Проверка
После `Developer: Reload Window` + новый чат — `ls -la` исчез, кириллика читается.

**Рабочая команда UTF-8:**
```powershell
$OutputEncoding = [System.Text.UTF8Encoding]::new(); [Console]::InputEncoding = [System.Text.UTF8Encoding]::new(); [Console]::OutputEncoding = [System.Text.UTF8Encoding]::new(); Get-ChildItem -Name
```
