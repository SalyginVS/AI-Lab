**Дата:** 2026-04-04
**Трек:** A (MCP Tools) + F (AI-as-Interface)
**Статус:** ✅ Завершён

## Задача

Создать custom MCP-сервер для семантического поиска по документации лаборатории (Markdown, Python, YAML, shell-скрипты). Второй MCP-сервер в стеке (после mcp-server-git), первый custom, первый серверный (Ubuntu-side), первый с HTTP-транспортом.

---

## Стек

- **MCP SDK:** mcp 1.27.0 (FastMCP, `mcp.server.fastmcp`)
- **Vector DB:** ChromaDB 1.5.5, PersistentClient, `~/rag-data/chroma/`
- **Embedding:** qwen3-embedding (4096d) через gateway /v1/embeddings
- **Транспорт:** streamable-http (порт 8100)
- **Deployment:** systemd (`rag-mcp.service`), UFW LAN-only
- **Корпус:** 133 файла / 286 chunks (gateway code, orchestrator, scripts, .md документация, ADR, грабли, результаты этапов)

---

## Архитектурные решения

### ADR-014: streamable-http для серверных MCP

**Контекст:** RAG MCP server нуждается в доступе к ChromaDB и документам на Ubuntu-сервере. Continue.dev MCP STDIO серверы запускаются на стороне клиента (Windows).

**Решение:** Использовать streamable-http транспорт. Сервер слушает на Ubuntu (порт 8100), Continue подключается по сети (`type: streamable-http`).

**Альтернативы отклонены:**
- STDIO — запустился бы на Windows, нет доступа к серверным ресурсам
- SSE — deprecated в MCP spec 2025-03-26, будет удалён

**Следствие:** Все будущие серверные MCP (Docker — этап 12) используют тот же паттерн: FastMCP + streamable-http + systemd.

## Data Path (подтверждён журналом gateway)

```
Continue Agent → search_docs tool call
  → RAG MCP Server (8100) → query embedding
    → gateway (8000) /v1/embeddings → Ollama → qwen3-embedding

```

## Грабли

### #38. ChromaDB collection.delete(where={}) несовместим с v1.5.5

**Симптом:** `indexer.py` падает при переиндексации на вызове `collection.delete(where={})`.
**Причина:** ChromaDB 1.5.5 не принимает пустой `where={}` для mass-delete.
**Решение:** Удалять коллекцию целиком (`chroma_client.delete_collection()`) и пересоздавать.

### #39. mcp.__version__ не работает в mcp 1.27.0

**Симптом:** `AttributeError: module 'mcp' has no attribute '__version__'`.
**Решение:** `from importlib.metadata import version; print(version("mcp"))`.

### #40. streamable-http требует Accept header

**Симптом:** MCP server отвечает ошибкой negotiation при curl без правильного Accept.
**Решение:** Обязательный заголовок: `Accept: application/json, text/event-stream`.
**Примечание:** Continue.dev делает это автоматически. Проблема только при ручном curl-тестировании.

### #41. MCP session lifecycle обязателен

**Симптом:** `tools/list` и `tools/call` без предварительного initialize возвращают ошибку.
**Решение:** Обязательная последовательность: initialize → notifications/initialized → tools/*. Во все запросы передавать `Mcp-Session-Id`.
**Примечание:** Continue.dev управляет lifecycle автоматически. Проблема только при ручном curl-тестировании.

### #42. UFW inactive despite saved rules

**Симптом:** `sudo ufw status` → `inactive`. Правила сохранены, но firewall не работает.
**Причина:** `ufw enable` не был выполнен ранее (или был отключён).
**Решение:** `sudo ufw enable`. Проверить: `sudo ufw status` → `active`.
**Критичность:** Высокая — все правила безопасности (порты 8000, 8100, 11434, 22) были неактивны.

## Стратегический результат

1. **Первый custom MCP-сервер** — паттерн: FastMCP + streamable-http + systemd. Тиражируется на Docker MCP (этап 12) и любые будущие серверные MCP.

2. **RAG pipeline end-to-end** — от Agent query через embedding до grounded ответа. Весь путь подтверждён логами gateway.

3. **Закрыт открытый вопрос #11** (ChromaDB vs Qdrant) — ChromaDB достаточен для PoC масштаба лаборатории (286 chunks, ~133 файла). Миграция на Qdrant при >100K docs или multi-user.

4. **Закрыт открытый вопрос #14** (chunking стратегия для русского) — Markdown-aware chunking + qwen3-embedding: кириллица работает не хуже английского (9/10 релевантных результатов на смешанном корпусе).

5. **Enterprise-переносимость:** паттерн RAG MCP → enterprise: замена ChromaDB на Qdrant, масштабирование корпуса, добавление reranking. Архитектурный каркас и data path идентичны.

6. **UFW починен** (Грабли #42) — безопасность всего стека теперь реально активна, не только декларативна.
