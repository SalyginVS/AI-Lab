## Drift-ы, обнаруженные и исправленные в ходе этапа

### #56. Ubuntu version: документация 22.04, факт 24.04

**Симптом:** setup-check.sh FAIL на проверке Ubuntu version.

**Факт:** Сервер работает на Ubuntu 24.04, все документы (паспорт, архитектура, инструкции) указывали 22.04.

**Влияние:** Не критично для runtime (стек работает на 24.04), но critical для documentation accuracy. Python 3.12+ (default в 24.04) объясняет `datetime.utcnow()` DeprecationWarning (Грабли #34).

**Действие:** Паспорт и архитектура обновляются на 24.04. setup-check.sh и ONBOARDING.md скорректированы.

### #57. Ollama override drift: пропавшие параметры

**Симптом:** setup-check.sh FAIL на `NUM_PARALLEL != 1`. Клиентский setup-check.ps1 FAIL на прямом Ollama path.

**Факт:** В `override.conf` отсутствовали `OLLAMA_NUM_PARALLEL=1` и `OLLAMA_HOST=0.0.0.0:11434`. Ollama слушал только 127.0.0.1 — FIM autocomplete с Windows не работал.

**Причина (A):** Вероятно, параметры были потеряны при одном из обновлений Ollama (upgrade script перезаписывает override.conf — Грабли #33). Либо были пропущены при ручном восстановлении.

**Решение:** Параметры восстановлены. Полный override теперь 7 переменных (OLLAMA_HOST, FLASH_ATTENTION, KV_CACHE_TYPE, MAX_LOADED_MODELS, NUM_PARALLEL, CUDA_VISIBLE_DEVICES, GGML_CUDA_NO_GRAPHS).

**Урок:** setup-check.sh оправдал своё существование в первом же прогоне — обнаружил real runtime drift, который не выявлялся health-check.sh (health-check проверяет liveness, не config).

### #58. Git MCP config naming: git.yml → git.yaml

**Симптом:** setup-check.ps1 FAIL на `mcpServers/git.yaml not found`.

**Факт:** Файл существовал как `git.yml`, а не `git.yaml`. Continue принимал оба расширения, но паспорт и check-скрипты ожидали `.yaml`.

**Решение:** Переименован в `git.yaml` для consistency.

### Многослойная модель ADR

| Слой | Описание | Кол-во | RAG-статус |
|------|----------|--------|-----------|
| L0 — Foundational | Базовые архитектурные принципы (Depth over Speed, Continue-first, STDIO, ChromaDB, Terminal Policy) | 5 | Primary index |
| L1 — Canonical | Текущие official architecture decisions (ADR-008…016) | 7 | Primary index |
| L2 — Subsidiary | Уточняющие/подчинённые решения (embeddings, planner, allowlist) | 11 | Backup (вне index) |
| L3 — Operational | Execution rules / implementation guardrails (commit-msg, MAX_LINES, venv) | 5 | Backup (вне index) |
| L9 — Closed | Закрытые/поглощённые решения | 1 | Backup (вне index) |
