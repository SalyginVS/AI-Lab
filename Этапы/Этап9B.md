# Этап 010 — Миграция Continue.dev embeddings → qwen3-embedding (9B)

## Цель

Переключить Continue.dev embeddings с локального [[transformers.js]] (all-MiniLM-L6-v2, 384 dim, VS Code process) на серверный [[qwen3-embedding]] (8B, 4096 dim, GPU) через [[gateway.py]] `/v1/embeddings`.

## Зависимости

- [[Этап009]] (9A) — gateway `/v1/embeddings` endpoint ✅

## Что сделано

- Embedding-блок в `config.yaml` переключён с `provider: transformers.js` на `provider: openai` через gateway
- `encoding_format: float` добавлен в `requestOptions.extraBodyProperties` — обязателен для совместимости
- `apiKey: ollama` — заглушка для openai provider
- `provider: codebase` добавлен в context-секцию (embeddings-backed retrieval)
- Gateway hotfix: `encoding_format` silent coerce вместо reject (400)
- Automatic reindex после cold restart VS Code подтверждён через логи gateway
- Documentation drift исправлен: Continue.dev 1.2.17 → 1.2.22

## Результат

Transport path подтверждён: Continue → gateway :8000/v1/embeddings → Ollama → qwen3-embedding → 200 OK батчами. FIM и chat не деградировали.
