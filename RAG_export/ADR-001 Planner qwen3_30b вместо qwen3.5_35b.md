# ADR-001: Planner = qwen3:30b вместо qwen3.5:35b

## Контекст

При реализации [[Этап08C]] (Orchestrator PoC) потребовалось выбрать модель для роли Planner в pipeline `plan-execute-review`. По исходному плану предполагался `qwen3.5:35b` как наиболее мощная reasoning-модель в лаборатории.

## Проблема

`qwen3.5:35b` нестабилен при загрузке через orchestrator.py в Ollama 0.18.0 на RTX 3090: возвращает 500/502 (`model failed to load`). Подробности: [[Грабли/01 qwen3.5:35b нестабилен как Planner в orchestrator]].

## Рассмотренные варианты

| Вариант | За | Против |
|---------|-----|--------|
| `qwen3.5:35b` | Лучший reasoning в лаборатории | Нестабилен при загрузке через orchestrator (500/502) |
| **`qwen3:30b`** | **Стабилен, вписывается в VRAM, хорошее планирование** | **Немного слабее по reasoning** |
| `deepseek-r1:32b` | Сильный reasoning | Уже занят ролью Reviewer; смешивать роли нежелательно |

## Решение

**Planner = `qwen3:30b`**

## Обоснование

- Стабильность важнее теоретического качества планирования: нерабочий pipeline бесполезен
- `qwen3:30b` показал качественные планы на всех трёх тестовых задачах (simple/medium/complex)
- Разделение ролей (qwen3:30b Planner / qwen3-coder:30b Executor / deepseek-r1:32b Reviewer) — чистая архитектура без дублирования моделей

## Статус решения

Принято и реализовано в [[pipelines.yaml]]. Пересмотр возможен при:
- Обновлении Ollama до версии, где qwen3.5:35b стабилен
- Отдельном PoC диагностики загрузки qwen3.5:35b

## Влияние на паспорт

[[Паспорт лаборатории v13]]: `qwen3:30b` переведён из Backup → **Active** (роль Planner).
