### Semantic Tool Pack v1

Был собран минимальный read-only tool pack из 5 инструментов: list_directory, find_files, read_text_file, get_git_status, get_changed_files. Все тесты через raw Ollama и gateway path.

**Ключевые находки:**

1. **Обе Gemma 4 поддерживают и semantic, и generic tool calling.** Старая гипотеза "Gemma = only semantic tools" опровергнута.

2. **gemma4:26b — лучший zero-shot semantic tool selector.** На gateway path стабильно выбирает правильный инструмент 3/3 без policy подсказок.

3. **gemma4:31b — quality-first, но чувствительна к tool selection.** Без policy prompt может выбрать более общий инструмент (list_directory вместо find_files). С минимальным system prompt ("Use the most specific tool available") стабилизируется 3/3.

4. **Multi-step semantic chaining работает.** Сценарий "найди файл → прочитай его" подтверждён для обеих моделей.

5. **Code и Review quality: Gemma 4 превосходит qwen3-coder:30b.** По code generation gemma4:31b лучшая, по review gemma4:26b лучшая (более дисциплинированная severity judgment).

## Результат: 10C-3 (Решение по ролям)

### ADR-012: Gemma 4 Model Integration

**Контекст:** Gemma 4 (31B Dense #3 Arena, 26B MoE #6 Arena) подтвердили function calling, multi-step chaining, thinking mode, vision и превосходство по code/review quality.

**Решение:** Не бинарное деление "Gemma semantic / Qwen generic", а ролевое распределение:

**Tier 1 — Primary workhorses:**

| Модель | Роль | Обоснование |
|--------|------|-------------|
| gemma4:31b | Quality-first Agent / Planner / Reviewer | #3 Arena, best code quality, thinking mode. Требует thin policy для tool selection |
| gemma4:26b | Fast Semantic Agent / Fast Executor | MoE speed, best zero-shot tool selector, strong review |
| qwen3-coder:30b | Stable Generic Executor / Backup Agent | Надёжнейший generic executor, backup при Gemma edge cases |

**Понижения:**
- qwen3:30b — Planner reserve (gemma4:31b забрала роль)
- glm-4.7-flash — Reserve (gemma4:26b занимает нишу)
- qwen3-vl:8b — Vision reserve (Gemma 4 имеют vision)
- deepseek-coder-v2:16b → **Deprecated** (роль полностью покрыта)

**Повышения:**
- qwen3.5:9b — tool calling восстановлен после Ollama 0.20.0

### Рекомендуемая ревизия pipelines.yaml

| Pipeline | Было | Рекомендация |
|----------|------|-------------|
| plan-execute-review | qwen3:30b → qwen3-coder:30b → deepseek-r1:32b | gemma4:31b → gemma4:26b → gemma4:31b |
| execute-review | qwen3-coder:30b → deepseek-r1:32b | gemma4:26b → gemma4:31b |
| review-only | deepseek-r1:32b | gemma4:31b |
| docs-generate | qwen3-coder:30b | gemma4:26b |
| commit-msg | qwen3.5:9b | qwen3.5:9b (без изменений) |

## Грабли

### Грабли #33: Ollama install.sh может затереть systemd override

**Симптом:** При апгрейде через `curl | sh` override.conf в `/etc/systemd/system/ollama.service.d/` может быть затёрт.

**Превентивное действие:**
```bash
# ДО апгрейда:
sudo cp /etc/systemd/system/ollama.service.d/override.conf ~/backup_10C/
# ПОСЛЕ апгрейда:
sudo cat /etc/systemd/system/ollama.service.d/override.conf
# Если пропал — восстановить из бэкапа
```
