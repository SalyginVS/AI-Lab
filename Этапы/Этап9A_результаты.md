## Цель этапа

Добавить в шлюз gateway.py новый endpoint `POST /v1/embeddings`, который проксирует запросы к Ollama `/api/embed` и возвращает ответ в формате OpenAI Embeddings API. Модель — `qwen3-embedding`. Backend-шаг, предшествующий переключению Continue.dev (этап 9B).

### Staged pipeline endpoint

Endpoint реализован как явный staged pipeline, без смешивания стадий:

1. **Parse request** — FastAPI/Pydantic разбирает тело запроса
2. **Semantic validation** — пустой input, allowlist моделей, dimensions, encoding_format
3. **Normalize** — input → внутренний `list[str]` (и single string, и batch)
4. **Build upstream payload** — сборка payload для POST /api/embed
5. **Call Ollama** — сетевой вызов к upstream
6. **Validate upstream response** — проверка embeddings, типов, cardinality
7. **Build OpenAI response** — формирование `data[index]`, usage, model

### Контракт endpoint

**Публичный минимальный контракт (вход):** `model`, `input` (str | list[str]), `dimensions` (опц.), `encoding_format` (опц.), `user` (опц.)

**Формат ответа (OpenAI-compatible):**
```json
{
  "object": "list",
  "data": [
    { "object": "embedding", "embedding": [...], "index": 0 }
  ],
  "model": "qwen3-embedding",
  "usage": { "prompt_tokens": 8, "total_tokens": 8 }
}
```

**Allowlist embedding-моделей:** только `qwen3-embedding` (расширяемо при добавлении новых моделей).

**Классы ошибок:** 400 (пустой input, unsupported model), 422 (schema validation), 502/503/500 (upstream errors).

## Принятые архитектурные решения

- **Additive change без модуляризации** — gateway.py остаётся монолитом; модуляризация перенесена на этап 10A (совместно с structured logging), чтобы не размывать scope 9A.
- **Batch support входит в 9A** — не откладывается, так как Ollama `/api/embed` поддерживает его нативно.
- **Allowlist на стороне gateway** — policy-layer не делегируется upstream. Ollama получает только pre-validated запросы.
- **Путь для Continue.dev через gateway** — explicit path: Continue → gateway `:8000/v1/embeddings` → Ollama `:11434/api/embed`. Прямой путь в Ollama не используется (в отличие от FIM autocomplete).
