### 8.3. Text-to-SQL PoC (Этап 11A) — Track F: AI-as-Interface

**Расположение:** `~/llm-gateway/scripts/text2sql_*.py`, `~/llm-gateway/ondo_demo.db`

**Назначение:** Benchmark NL→SQL (Natural Language → SQL) на локальном контуре. Первый PoC паттерна AI-as-Interface.

**Стек:**
- SQLite (`ondo_demo.db`): 6 таблиц, 171 запись, схема ONDO (departments, employees, facilities, energy_consumption, hotel_bookings, production_orders)
- Python-скрипт через OpenAI SDK → gateway → Ollama
- Read-only enforcement: SQLite `PRAGMA query_only = ON` + regex-валидация
- 20 тестовых вопросов (4 уровня: simple, join+groupby, subquery, edge cases + safety + hallucination)

**Скрипты:**

| Скрипт | Назначение | Evaluator |
|--------|-----------|-----------|
| `text2sql_poc.py` | Strict evaluator (v1) | Формальное совпадение |
| `text2sql_poc_v2.py` | Business-aware evaluator (v2) | Note-aware, семантический |
| `text2sql_poc_v3.py` | v2 + prompt addendum | Note-aware + prompt engineering |

**Benchmark results (baseline truth, v2 evaluator):**

| Модель | EA | SA | Latency |
|--------|----|----|---------|
| **qwen3-coder:30b** | 100% | 75% | 0.4s |
| gemma4:31b | 95% | 70% | 4.0s |
| gemma4:26b | 95% | 65% | 1.5s |

**Best observed (v3, + prompt addendum):**

| Модель | EA | SA | Дельта |
|--------|----|----|--------|
| qwen3-coder:30b | 100% | 90% | +15 п.п. |
| gemma4:31b | 100% | 90% | +20 п.п. |
| gemma4:26b | 100% | 85% | +20 п.п. |

**Ключевые выводы:**
- NL→SQL жизнеспособен на локальных моделях
- qwen3-coder:30b — лучший SQL executor (speed + accuracy)
- Prompt engineering: +15–20 п.п. SA (model-agnostic), но потолок ~90% без semantic layer
- Safety/hallucination: 100% pass rate (все модели, все версии)
- Prompt bleed при micro-tuning — retrieval-based approach нужен для >90%

**Статус:** PoC завершён. Следующий шаг: semantic grounding / retrieval-assisted SQL.
