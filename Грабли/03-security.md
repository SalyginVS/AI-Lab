## Security Policy

### Секреты и credentials

- НИКОГДА не читай, не выводи, не модифицируй файлы с секретами: `.env`, `**/secrets/**`, `**/.ssh/**`, `**/id_rsa*`, `**/*.pem`, `**/*.key`.
- НИКОГДА не выполняй команды, выводящие содержимое переменных окружения с токенами: `$LLM_GATEWAY_TOKENS`, `$LLM_ORCHESTRATOR_TOKEN`, `$env:LLM_GATEWAY_TOKENS`, `$env:LLM_ORCHESTRATOR_TOKEN`.
- В примерах кода и конфигурациях используй заглушки: `sk-lab-EXAMPLE`, `Bearer sk-lab-EXAMPLE`. Не подставляй реальные значения токенов.
- При обнаружении утечки секрета (токен в stdout, в коммите, в логе) — немедленно сообщи пользователю. Не пытайся исправить самостоятельно.

### Denylist путей

Запрещены операции чтения, записи, удаления для следующих путей:

- `~/llm-gateway/.env`
- `~/.env`
- `~/.ssh/`
- `~/rag-mcp-server/.env`
- `/etc/systemd/system/ollama.service.d/override.conf` (только через SOP Ollama Upgrade)

При необходимости работы с этими файлами — запроси явное подтверждение пользователя и объясни, что именно будет сделано.

### Git safety

- Перед `git add` или `git commit` проверь, что staged файлы не содержат строк с паттернами: `sk-lab-`, `Bearer sk-lab`, реальных hex-токенов длиной 40+ символов.
- Никогда не коммить `.env` файлы. Если `.gitignore` не содержит `.env` — предупреди пользователя.

### Сетевые операции

- Не выполняй curl/wget запросы к внешним хостам без явного запроса пользователя. Вся работа — внутри LAN (192.168.0.0/24) и WireGuard (10.10.10.0/24).
- При формировании curl-запросов к gateway (порт 8000) используй `$TOKEN` переменную, не вставляй литеральные токены в команду.
