# Этап 10A — Structured JSON Logging + Модуляризация gateway

## Суть

Монолит `gateway.py` v0.8.0 (1305 строк) разбит на Python-пакет `gateway/` (8 модулей, 1793 строки). Добавлен structured JSON logging — каждый LLM-запрос логируется как JSON event в systemd journal с полями tokens, latency_ms, ttft_ms, model, request_id.

## Два подэтапа

- **10A-1** — модуляризация: пакет `gateway/`, APIRouter, `run.py` entrypoint, обновление systemd
- **10A-2** — structured logging: Custom JSONFormatter (stdlib), события `llm_completion` + `llm_embedding`, uvicorn.access отключён

## Ключевые решения

- Formatter: Custom JSONFormatter (stdlib, ноль зависимостей) вместо python-json-logger или structlog
- Access log: Схема C — uvicorn.access отключён, свой middleware вместо stock
- Иерархия импортов без циклов: `__init__` → `models` → `errors` → `upstream` → `chat/embeddings` → `app`
- ADR-008 закрыт (модуляризация выполнена в срок)

## Результат

- gateway v0.9.0, 8 модулей
- `journalctl -o cat | jq` парсит все строки
- ttft_ms в stream events (176ms горячая, 2871ms холодная)
- Docstring исправлен: MAX_LOADED_MODELS=2
