# Этап 8D — Headless Automation PoC

## Цель

Превратить [[orchestrator.py]] из интерактивного PoC в инфраструктурный инструмент слоя 4 (Orchestration/Automation), вызываемый из shell-скриптов и git-хуков без IDE и без человека в контуре.

## Что сделано

- `orchestrator.py` → **v1.1.0**: добавлен флаг `--stdout`, исправлен `load_pipelines()` под схему `pipelines:` в YAML
- `pipelines.yaml`: добавлен pipeline `commit-msg` (qwen3.5:9b, 1 шаг, быстрый)
- `scripts/auto-review.sh`: code review по git diff (режимы: `--staged` / `--last-commit` / `--branch`)
- `scripts/auto-commit-msg.sh`: генерация conventional commit message для staged diff
- `scripts/auto-docs.sh`: документация файла (форматы: `docstring` / `markdown`)
- `.githooks/pre-push`: автоматический review перед push, STRICT_REVIEW с fail-closed в non-TTY

## Паттерн

```
shell (context capture) → orchestrator.py --stdout → stdout текст
```

Единственная зависимость скриптов от среды — путь к `venv`. В Enterprise-варианте заменяется HTTP-вызовом к gateway API.

## Подтверждённые сценарии (7/7)

| Сценарий | Результат |
|----------|-----------|
| auto-review --last-commit | Review в stdout, exit 0 |
| auto-commit-msg (staged) | Строка `docs(README_8D4): add...`, exit 0 |
| auto-docs gateway.py --format markdown | Структурированная markdown-документация |
| pre-push non-strict | Review + push прошёл |
| pre-push STRICT_REVIEW=1 non-TTY | Review + exit 1 (fail-closed) |
| auto-review (пустой diff) | `No changes to review`, exit 0 |
| auto-commit-msg (нет staged) | ERROR + exit 1 |
