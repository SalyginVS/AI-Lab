# Этап 9B — Миграция Continue.dev embeddings → qwen3-embedding через gateway

**Статус:** ✅ Завершён  
**Дата:** 2026-03-29  
**Исполнители:** ChatGPT (реализация config, диагностика) + Claude (планирование, финализация, gateway patch) + Владимир (деплой и верификация)  
**Паспорт после этапа:** v16.0

---

## Цель этапа

Переключить Continue.dev embeddings с локального transformers.js (all-MiniLM-L6-v2, 384 dimensions, преимущественно английская модель, работает внутри VS Code process) на серверный qwen3-embedding (8B, до 4096 dimensions, 100+ языков, GPU) через gateway `/v1/embeddings`. Backend endpoint уже реализован в этапе 9A (gateway v0.8.0).

---

## Что было сделано

### Изменения в config.yaml (клиент)

Embedding-блок заменён с:

```yaml
- name: embedder
  provider: transformers.js
  model: all-MiniLM-L6-v2
  roles:
    - embed
```

На:

```yaml
- name: qwen3-embedding
  provider: openai
  model: qwen3-embedding
  apiBase: http://192.168.0.128:8000/v1
  apiKey: ollama
  roles:
    - embed
  embedOptions:
    maxChunkSize: 512
    maxBatchSize: 5
  requestOptions:
    timeout: 60000
    extraBodyProperties:
      encoding_format: float
```

### Добавлен context provider

В секцию `context:` добавлен `provider: codebase` — embeddings-backed retrieval по workspace. До 9B его не было (11 providers, теперь 12).

**Примечание:** `provider: codebase` помечен как deprecated в документации Continue. Рекомендуемая замена — Agent mode built-in tools. Оставлен для backward compatibility и как потребитель embedding-индекса.

### Gateway hotfix (сервер)

В `gateway.py` v0.8.0 исправлена валидация `encoding_format`: вместо reject (400) при `encoding_format != "float"` — silent coerce to float. Контракт не изменился (gateway по-прежнему возвращает только float embeddings). Это bug fix, не bump версии.

**До:**
```python
if req.encoding_format is not None and req.encoding_format != "float":
    raise GatewayError(400, "Only encoding_format='float' is supported", ...)
```

**После:**
```python
if req.encoding_format is not None and req.encoding_format != "float":
    req.encoding_format = "float"
```

**Верификация hotfix:** `POST /v1/embeddings` с `encoding_format="base64"` → 200 OK, `data_len=1`, `emb_len=4096`.

## Подтверждённые результаты

### Transport path

| Тест | Метод | Результат |
|------|-------|-----------|
| Gateway embedding single | `curl POST /v1/embeddings input="test"` | 200 OK, embedding length = 4096 |
| Gateway embedding batch | `curl POST /v1/embeddings input=["a","b"]` | 200 OK, data_len = 2 |
| encoding_format coerce | `curl POST /v1/embeddings encoding_format="base64"` | 200 OK (coerced to float) |
| Continue → gateway reindex | Cold restart VS Code, journalctl -u llm-gateway | Серии `POST /v1/embeddings 200` в логах |
| Index exists | `%USERPROFILE%\.continue\index\` | Каталог непустой после reindex |

### Отсутствие деградации

| Тест | Результат |
|------|-----------|
| FIM autocomplete | Работает без заметной деградации |
| Chat после индексации | Работает (холодный старт chat-модели допустим) |

### Retrieval quality (@Codebase)

| Тест | Результат | Комментарий |
|------|-----------|-------------|
| Symbol lookup `build_openai_response` | Частичный hit | Continue находил правильный файл по имени, но содержимое цитаты не всегда совпадало с ground truth |
| EN semantic | Нестабильный | Иногда grounded, иногда synthetic |
| RU semantic | Слабый | Часто generic generation вместо project-grounded |

**Важно:** `@Codebase` context provider помечен как **deprecated** в документации Continue. Слабые результаты retrieval **не являются доказательством** плохого quality qwen3-embedding — они отражают состояние deprecated retrieval pipeline в Continue 1.2.22. Embedding quality нужно валидировать изолированно (direct similarity через curl) или через Agent mode built-in tools.

## Архитектурные решения

1. **Gateway-first path для embeddings.** Continue → gateway `:8000/v1/embeddings` → Ollama. Прямой путь в Ollama не используется (в отличие от FIM autocomplete). Embeddings проходят через тот же policy/logging layer, что и chat.

2. **provider: openai, не provider: ollama.** Выбран `provider: openai` с `apiBase` на gateway, а не `provider: ollama` с прямым подключением. Причина: единый policy layer, logging, allowlist, потенциальные метрики.

3. **encoding_format coerce в gateway.** Вместо reject (400) — silent coerce to float. Defense in depth: клиент тоже явно отправляет `encoding_format: float`.

4. **provider: codebase добавлен в context.** Необходим как потребитель embedding-индекса для `@Codebase`. Deprecated, но функционален.

## Баги и грабли

| # | Грабля | Вывод |
|---|--------|-------|
| 1 | **encoding_format: float обязателен** | Continue `provider: openai` отправляет encoding_format, несовместимый с gateway. Fix: `requestOptions.extraBodyProperties.encoding_format: float` в config + gateway hotfix (silent coerce). |
| 2 | **apiKey заглушка обязательна** | Continue `provider: openai` требует непустое поле `apiKey`. При отсутствии — запросы не отправляются. Fix: `apiKey: ollama` (или любая строка). Gateway auth опциональна, значение игнорируется. |
| 3 | **Windows EBUSY при Rebuild codebase index** | Штатная команда `Continue: Rebuild codebase index` не может удалить `index.sqlite` (файл заблокирован процессом VS Code). Fix: полный cold restart VS Code + при необходимости ручное переименование `%USERPROFILE%\.continue\index\`. Automatic reindex при cold restart работает надёжно. |
| 4 | **@Codebase deprecated** | `provider: codebase` помечен deprecated в документации Continue. В актуальных версиях рекомендуется Agent mode built-in tools. Deprecated provider может давать нестабильные результаты (synthetic code вместо grounded retrieval). Не использовать как benchmark для embedding quality. |
| 5 | **Documentation drift: Continue version** | Фактическая версия Continue в стеке — 1.2.22. В документации фигурировала 1.2.17. Разница: 5 minor releases, changelog не проверялся на breaking changes. |

## Что сознательно не делалось в 9B

- Модуляризация gateway.py — перенесена на 10A
- Structured logs embedding-запросов — этап 10A
- Глубокая A/B-оценка embedding quality (qwen3-embedding vs transformers.js) — требует изолированной методики (direct similarity или Agent mode), не через deprecated @Codebase
- Tuning параметров: maxChunkSize, MRL dimension truncation, reranking — за рамками integration-этапа
- Исследование Continue retrieval pipeline internals — отдельная диагностическая задача

## Методологический урок этапа

**@Code и @Codebase — это разные механизмы.** `@Code` (provider: code) использует tree-sitter/AST по открытым файлам, не зависит от embeddings. `@Codebase` (provider: codebase) использует embeddings vector search по всему workspace. Baseline-тесты до миграции делались через `@Code`, post-migration тесты — через `@Codebase`. Это не корректный A/B, а сравнение двух разных retrieval контуров.

**Не тестировать новый backend через deprecated frontend.** @Codebase deprecated — его поведение не отражает quality embeddings. Для изоляции embedding quality нужен либо direct curl к gateway (embedding similarity), либо инспекция index.sqlite, либо Agent mode built-in tools.
