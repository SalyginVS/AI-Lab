### 1.2. Перезапуск компонентов

```bash

# RAG MCP (зависит от gateway)
sudo systemctl restart rag-mcp

# Docker MCP (зависит от Docker)
sudo systemctl restart docker-mcp
```

### 1.3. Проверка здоровья

```bash
# Быстрая проверка
curl -s http://localhost:8000/health | jq .

# Полный health-check (11 проверок)
bash ~/llm-gateway/scripts/health-check.sh

# Setup-check (66 проверок)
bash ~/llm-gateway/scripts/setup-check.sh
```

### 1.4. Systemd override Ollama

Файл: `/etc/systemd/system/ollama.service.d/override.conf`

```ini
[Service]
Environment="OLLAMA_HOST=0.0.0.0:11434"
Environment="OLLAMA_FLASH_ATTENTION=1"
Environment="OLLAMA_KV_CACHE_TYPE=q8_0"
Environment="OLLAMA_MAX_LOADED_MODELS=2"
Environment="OLLAMA_NUM_PARALLEL=1"
Environment="CUDA_VISIBLE_DEVICES=0"
Environment="GGML_CUDA_NO_GRAPHS=1"
```

После редактирования:
```bash
sudo systemctl daemon-reload
sudo systemctl restart ollama
```

**GGML_CUDA_NO_GRAPHS=1** — обязателен. Без него Agent mode (каскад tool-call запросов) вызывает CUDA graph capture crash (SIGABRT).

### 1.5. Systemd gateway service

Файл: `/etc/systemd/system/llm-gateway.service` — использует `EnvironmentFile=~/llm-gateway/.env` для загрузки токенов. Тот же `.env` расшарен с `rag-mcp.service`.

## 11. Диагностика проблем

### 11.1. Gateway не стартует

```bash
journalctl -u llm-gateway -n 50 --no-pager
# Частая причина: отсутствует .env или неправильный формат токенов
```

### 11.2. Модель не загружается

```bash
ollama ps                              # Что загружено
journalctl -u ollama -n 20 --no-pager  # Логи Ollama
nvidia-smi                             # Память GPU
```

### 11.3. Continue не видит модели

1. Проверить gateway: `curl -s http://192.168.0.128:8000/v1/models -H "Authorization: Bearer $TOKEN" | jq .`
2. Проверить apiKey в config.yaml (должен быть реальный sk-lab-... токен)
3. Ctrl+Shift+P → Developer: Reload Window

### 11.4. MCP tools не работают

1. MCP работает **ТОЛЬКО в Agent mode** (не Chat, не Edit)
2. SSH tunnel для Docker MCP: `ss -ltnp | grep 8200`
3. После поднятия tunnel: Developer: Reload Window
4. Проверить MCP: Continue → Tools → MCP Servers (зелёный статус)

### 11.5. Autocomplete не работает

1. Autocomplete идёт **напрямую в Ollama** (не через шлюз)
2. Проверить: `curl http://192.168.0.128:11434/api/tags | jq .`
3. Timeout должен быть 30000ms (холодный старт qwen2.5-coder:7b = 5-15 сек)
