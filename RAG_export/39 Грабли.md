# Грабли 36 — Prompt bleed при точечном tightening NL-to-SQL rules

**Симптом:** После добавления точечного rule для booking revenue (exclude cancelled) в prompt addendum, Q10 исправился, но общий SA упал с 90% до 85% — regression на других кейсах.

**Причина:** Монолитный prompt addendum создаёт coupling между rules. Усиление одного rule меняет поведение модели на не связанных вопросах (over-generalization / prompt bleed).

**Решение:** Остановить micro-tuning. Зафиксировать v3 до booking patch как best observed. Дальнейшее улучшение — через retrieval-based approach (schema-grounded few-shot examples) вместо монолитного prompt.

**Урок для enterprise:** Prompt engineering для NL-to-SQL не масштабируется линейно. После ~90% SA нужен semantic layer / retrieval-assisted SQL, а не бесконечное добавление rules в system prompt.
