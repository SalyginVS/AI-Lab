# 42 UFW inactive despite saved rules

## Симптом

`sudo ufw status` → `inactive`. Правила для портов 8000, 8100, 11434, 22 были сохранены, но firewall не работал.

## Причина

`ufw enable` не был выполнен после настройки правил (или был отключён ранее).

## Решение

```bash
sudo ufw enable
sudo ufw status  # → active
```

## Критичность

Высокая. Все правила безопасности (ограничение доступа к портам только из LAN) были неактивны. Сервер был открыт для любого сетевого доступа.
