## 2. MCP Servers

### 2.1. Git (STDIO, клиентская сторона)

Файл: `%USERPROFILE%\.continue\mcpServers\git.yaml`

```yaml
name: Git MCP Server
version: 0.0.1
schema: v1

mcpServers:
  - name: git
    type: stdio
    command: uvx
    args:
      - "mcp-server-git"
```

12 tools. Требует `uv/uvx` (Astral) v0.10.12+ на Windows.

### 2.2. RAG (streamable-http, серверная сторона)

```yaml
schema: v1

  - name: rag
    type: streamable-http
    url: http://192.168.0.128:8100/mcp
```

5 tools: search_docs, get_doc_section, grep_doc, list_collections, index_status.

### 2.3. Docker (streamable-http через SSH tunnel)

Файл: `%USERPROFILE%\.continue\mcpServers\docker.yaml`

```yaml
name: Docker MCP Server
version: 0.0.1
schema: v1

mcpServers:
  - name: docker
    type: streamable-http
    url: http://localhost:8200/mcp
```

10 tools. **Требует SSH tunnel:** `ssh -L 8200:127.0.0.1:8200 user@192.168.0.128`

### 2.4. MCP YAML format (Грабли #60)

```yaml
# Правильно — плоская структура:
- name: rag
  type: streamable-http
  url: http://192.168.0.128:8100/mcp

# НЕПРАВИЛЬНО — вложенный transport:
- name: rag
  transport:
    type: streamable-http
    url: ...
```

## 5. Полный список грабель Continue (v1.2.22)

### Конфигурация

| # | Грабля | Решение |
|---|---|---|
| 1 | timeout в миллисекундах | 600000 = 10 мин (chat), 30000 = 30 сек (autocomplete) |
| 2 | extraBodyProperties ВНУТРИ requestOptions | Не на уровне модели — иначе 400 от Ollama |
| 3 | YAML anchors не принимает | Копировать полные блоки |
| 4 | Имена моделей со скобками = Invalid input | Использовать простые имена (qwen35, gemma4-31b) |
| 5 | roles: agent невалидна | Agent mode через UI selection, не через roles |
| 6 | tabAutocompleteModel не существует в YAML | Модель с roles: [autocomplete] в models |
| 7 | debounce → debounceDelay | Правильное имя параметра |
| 8 | Секция context: сбрасывает дефолты | Перечислять все провайдеры явно |
| 9 | Проектные rules требуют Reload Window | Developer: Reload Window |
| 10 | capabilities: [tool_use] не нужен | Continue автодетектит |

### MCP

| # | Грабля | Решение |
|---|---|---|
| 11 | MCP работает ТОЛЬКО в Agent mode | Не в Chat, не в Edit |
| 12 | Плоская структура YAML (Грабли #60) | type: + url:, не вложенный transport: |
| 13 | MCP reconnect после SSH tunnel (Грабли #63) | Developer: Reload Window |

### Embeddings

| # | Грабля | Решение |
|---|---|---|
| 14 | encoding_format: float обязателен | В extraBodyProperties |
| 15 | apiKey заглушка обязательна | Указать реальный токен |
| 16 | Rebuild codebase index → EBUSY на Windows | Перезапустить VS Code |
| 17 | @Codebase provider deprecated | Использовать Agent built-in tools |

### Диагностика

| # | Действие | Как |
|---|---|---|
| 1 | Ошибки конфига | Правый нижний угол VS Code ⚠ Continue |
| 2 | Console логи | DevTools → Console → фильтр continuedev |
| 3 | Сетевые запросы | **НЕ видны** в Network tab (extension host) |
| 4 | Open Config | «Continue: Open Settings» (не «Open Config File») |
| 5 | Reload | Ctrl+Shift+P → Developer: Reload Window + новый чат |
