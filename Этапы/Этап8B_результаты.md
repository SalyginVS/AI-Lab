## Что сделано

### 8B-1 — Диагностика проблемы

Обнаружена проблема: Continue Agent в Windows иногда генерирует bash-style команды (`ls -la`, `cat`, флаги вида `-la`) вместо PowerShell-native. Это вызывает ошибки выполнения, так как PowerShell не понимает bash flags.

Дополнительно выявлена проблема с кодировкой: файлы с кириллическими именами выводятся кракозябрами при `Get-ChildItem` в стандартной PowerShell-сессии. Причина — runtime-кодировка PowerShell по умолчанию ≠ UTF-8 (Unicode Transformation Format, 8 бит).

### 8B-2 — Реализация через global rule

Исправление реализовано через добавление Windows-блока в `01-general.md` (`alwaysApply: true`). Бэкенд (Ollama, gateway.py, Ubuntu) не менялся.

**Добавленные правила (terminal block в `01-general.md`):**

```markdown
## Terminal (Windows / PowerShell)

- Используй PowerShell-native команды: Get-ChildItem, Get-Content, Where-Object, Select-String и т.д.
- Не используй bash-style флаги: ls -la, cat, grep и аналоги недоступны нативно.
- Каждая команда должна быть self-contained — не полагайся на состояние предыдущих команд.
- Не создавай временные файлы без явного запроса пользователя.
- Не оборачивай всю команду в строковые кавычки.
- Для файлов с кириллицей и другими non-ASCII именами перед directory listing / file read
  выставляй UTF-8:
  $OutputEncoding = [System.Text.UTF8Encoding]::new();
  [Console]::InputEncoding = [System.Text.UTF8Encoding]::new();
  [Console]::OutputEncoding = [System.Text.UTF8Encoding]::new();
```

### 8B-3 — Проверка результата

После `Developer: Reload Window` + новый чат:

- Agent перестал использовать `ls -la` → переключился на `Get-ChildItem`, `Get-Content`, `Where-Object`
- Кириллические имена файлов отображаются корректно при использовании UTF-8 преамбулы

**Подтверждённые рабочие имена файлов после UTF-8 преамбулы:**
- `Грабли`, `Компоненты`, `Конфиги`, `Модели`, `Решения`, `Этапы`
- `Домашняя_ИИ_лаборатория_Continue_инструкция_p1.md`
- `Паспорт_лаборатории_v11.md`, `Целевая_архитектура_AI_Coding_Platform_v1.3.md`

**Рабочая команда для кириллицы (подтверждено на стенде):**
```powershell
$OutputEncoding = [System.Text.UTF8Encoding]::new(); [Console]::InputEncoding = [System.Text.UTF8Encoding]::new(); [Console]::OutputEncoding = [System.Text.UTF8Encoding]::new(); Get-ChildItem -Name
```
