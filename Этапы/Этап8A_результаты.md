## Обнаруженные проблемы и грабли

### 1. uvx PATH-эффект на Windows (НИЗКО)

**Симптом:** после `winget install astral-sh.uv` команда `uvx` не найдена в текущем терминале.

**Причина:** Windows не обновляет PATH в запущенных сеансах.

**Решение:** перезапустить VS Code / PowerShell. Или закрыть и открыть терминал.

### 2. PowerShell && разделитель (НИЗКО)

**Симптом:** в первом tool call `git_status` внутренний шаг `pwd && git status` дал ошибку парсинга.

**Причина:** `&&` — не валидный разделитель команд в PowerShell (только в cmd.exe и bash). Это поведение mcp-server-git или встроенного Continue terminal.

**Решение:** Continue корректно продолжил с чистым `git status`. Не блокер, но стоит учитывать для диагностики.

### 3. git add . по умолчанию (СРЕДНЕ, для боевого использования)

**Симптом:** при запросе "stage all changes" агент выполняет `git add .` — включая всё, что в директории.

**Причина:** модель интерпретирует "all changes" буквально.

**Митигация:** для критичных репозиториев — добавить правило в rules: "Перед git add показать список файлов и запросить подтверждение" или "Никогда не делать git add . — только именованные файлы".

## Поток данных MCP (подтверждённый)

```
Пользователь: "Show me git status"
     │
     ▼
Continue Agent mode → gateway.py :8000 → Ollama (qwen3-coder:30b)
                                            │
                                            ▼ tool_call: git_status
                                            │
Continue Agent ← gateway.py ← Ollama ──────┘
     │
     ▼ вызов MCP Server
Continue Agent → STDIO → uvx mcp-server-git → git CLI (Windows)
     │                                            │
     ▼ tool_result                                │
Continue Agent ← STDIO ← mcp-server-git ←────────┘
     │
     ▼ отправка tool_result
Continue Agent → gateway.py :8000 → Ollama → финальный ответ
     │
     ▼
Пользователь видит ответ
```

**Ключевое:** gateway.py участвует в LLM-запросах (генерация tool_call и финальный ответ). MCP-коммуникация (STDIO) идёт напрямую между Continue и mcp-server-git, минуя gateway.
