# Этап 9A — gateway.py v0.8.0: /v1/embeddings

## Цель

Добавить `POST /v1/embeddings` в [[Компоненты/gateway|gateway.py]] как OpenAI-compatible проксирование к `POST /api/embed` Ollama. Backend-шаг перед [[Этапы/Этап010|9B]] (миграция Continue.dev).

## Что реализовано

- Новый endpoint `POST /v1/embeddings` — additive change, chat-path не затронут
- Single input и batch input (`input: str | list[str]`)
- Staged pipeline: parse → semantic validation → normalize → build payload → call Ollama → validate response → build OpenAI response
- Allowlist embedding-моделей: только `qwen3-embedding`
- Классы ошибок: 400 (empty/unsupported), 422 (schema), 502/503/500 (upstream)
- [[Компоненты/gateway|gateway.py]] v0.7.0 → **v0.8.0**

## Подтверждённые тесты

- [x] Happy path: single input → 200, OpenAI-style response
- [x] Happy path: batch input `["alpha","beta"]` → 200, data_count=2, indexes=[0,1]
- [x] Negative: empty input `""` → 400, `empty_input`
- [x] Negative: schema `input=[1,2]` → 422, `validation_error`
- [x] Negative: unsupported model `qwen3:30b` → 400, `unsupported_embedding_model`

## Архитектурные решения

- [[Решения/ADR-008 Embeddings без модуляризации|ADR-008]] — additive change без модуляризации; модуляризация → 10A
- [[Решения/ADR-009 Allowlist embedding-моделей|ADR-009]] — allowlist на стороне gateway, не делегируется upstream
- Путь для Continue.dev: через gateway (`:8000/v1/embeddings`), не напрямую в Ollama
