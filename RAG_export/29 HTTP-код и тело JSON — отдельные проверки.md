# 25 HTTP-код и тело JSON — отдельные проверки

## Проблема

При проверке негативных сценариев легко смотреть только на тело ответа (`jq`) и считать тест пройденным, если JSON-структура правильная. Но FastAPI может вернуть корректный JSON error-body с неправильным HTTP-кодом.

## Пример

```bash
# НЕПОЛНАЯ проверка — видим только тело
curl -s http://localhost:8000/v1/embeddings \
  -d '{"model":"qwen3-embedding","input":""}' | jq

# ПОЛНАЯ проверка — тело И код отдельно
curl -s http://localhost:8000/v1/embeddings \
  -d '{"model":"qwen3-embedding","input":""}' | jq
# →  {"error": {"code": "empty_input", ...}}

curl -s -o /dev/null -w "%{http_code}\n" \
  http://localhost:8000/v1/embeddings \
  -d '{"model":"qwen3-embedding","input":""}'
# → 400  ← это и нужно проверять отдельно
```

## Правило

Для каждого негативного теста — **две команды**: одна на тело (jq), вторая на HTTP-код (`-w "%{http_code}"`). HTTP semantics — часть контракта, не опциональная проверка.
